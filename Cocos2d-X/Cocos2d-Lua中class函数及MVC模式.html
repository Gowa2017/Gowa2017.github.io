Cocos2d-Lua中class函数及MVC模式

留言

Apr 01, 2019Cocos2d-X点击

Lua 本身是不提供继承的，但是由于每个对象（表） 都会有一个 元表，根据我们条件不不同可能会触发对应的元方法。在 lua中的表及面向对象 中有相关的描述。

class 函数

Cocos2d-Lua 中定义了一个 class 用来实现继承，更继以实现一些面向对象的编程方法。我们可以看看其代码的实现。

首先会建立一个 cls 对象，并将其字段 __cname 设置为类名。

cls 后续的参数数量不定。但参数参数的类型只能是 nil, table, function 三种。

  • 如果参数类型是 function，那么就将 cls 的函数 __create() 设置为此 function。
  • 如果参数的类型是 table，这有两种可能。一是此 table 的 .isclass 字段存在，表示是一个原生的 c 类，那么就将 cls 的构造函数 __create() 返回值，设置为此 table 中的 create() 返回值；如果此 table 不是一个原生类，将其放到 cls 的 __supers 字段中，接着，将第一个 lua class 设置为父类。

最后，如果父类为空，或者只有一个父类，相当于把 cls 的元表设置为 父类。否则的话就会遍历所有的父类来查找对应的字段（成员）

如果 cls 的 构造器 ctor 不存在，那么就设置一个空函数。

定义 cls.new() 方法，这个方法会从 __create() 方法获取对象，接着把对象的元方法设置为 cls。然后再调用 ctor。

我们来看一下文中提到的几个方法：

  • cls.create() 实际上就是调用了 cls.new(…) 而已
  • cls.new() 调用 cls.__create()() 来获取对象，调用 cls:ctor() 来进行构造。
  • cls.__create()() 这个方法，要么指向一个函数（函数作为父类）；要么是对原成类 create() 方法的封装（父类是原生类）；要么为空（父类是 lua 类）。
  • cls:ctor() 这个是一个构造函数。当 __create()() 为空的时候，就靠这个方法来进行初始化了。

AppBase

AppBase 是一个 Lua 类，其没有父类。

我们会将自己的 App 类继承 AppBase，在 onCreate() 内实现初始化逻辑。

require("app.MyApp"):create():run() 建立一个 MyApp 对象，然后运行。实际上就是进入默认场景：

对于场景的切换分了两步：

  1. 建立 View。查找 viewsroot 下的 lua 文件来找到对应的视图文件，用 require 进行加载，调用其 view:create(self, name) 进行创建。
  2. 调用 View:showWithScene() 方法。

MVC

ViewBase

所有的 Scene 类都继承至 ViewBase。 ViewBase 是一个继承至原生类 cc.Node。

我们知道， cc.Node 是 场景图 的一个基本元素，其最常见的子类是 Scene, Layer, Menu, Sprite, Label：

Node is the base element of the Scene Graph. More…
Inherits Ref.
Inherited by __NodeRGBA, AtlasNode, AttachNode, BaseLight, Camera, ClippingNode, ClippingRectangleNode, DrawNode, TMXLayer, TMXTiledMap, TableViewCell, GLNode, Label, Layer, MenuItem, MotionStreak, MotionStreak3D, NodeGrid, ParallaxNode, ParticleBatchNode, ParticleSystem, ProgressTimer, ProtectedNode, RenderTexture, Scene, Skybox, Sprite, Sprite3D, SpriteBatchNode, Terrain, TMXTiledMap, and GLNode.

所以事实上 ViewBase 返回的是一个 cc.Node 原生对象。所以其 __create()() 方法不为空。

当 App:createView(self, name) 的时候，都会传入 app 这个实例及视图名称。

事实上，场景的切换，最终都会走到 app:enterScene[Ex]() 来。

Node

主要功能：

  • 包含其他 Node 对象( addChild, getChildByTag, removeChild等）
  • 调度定时任务(schedule, unschedule等）
  • 执行动作（runAction, stopAction等）

继承一个子 Node 通常意味着：

  • 重写 init 来初始化资源和调度回调
  • 建立回调来更好的控制时间
  • 重写 draw() 来渲染接点。

属性列表：

  • position (default: x=0, y=0)
  • scale (default: x=1, y=1)
  • rotation (in degrees, clockwise) (default: 0)
  • anchor point (default: x=0, y=0)
  • contentSize (default: width=0, height=0)
  • visible (default: true)

Layer

Layer 继承自 Node，然后实现了 TouchEventsDelegate 协议。

  • 可以接收 iPhone 触摸事件
  • 可以接收加速器输入

调用 Layer.create() 方法会创建一个全屏的黑色 layer。

BaseFrame Model

所有的数据模型都继承这个类，这个类也是一个 Lua 类。

其定义了一些比较通用的方法，主要是从网络获取数据的模型。

  • onCreateSocket() 建立套接字。
  • onSocketCallBack() 网络消息回调，处理一些连接事件与网络消息。网络消息交给函数 onSocketEvent() 与 onGameSocketEvent() 处理。
  • onSocketEvent() 空函数。需要子类重写来处理消息逻辑。
  • onGameSocketEvent() 空函数。需要子类重写来处理消息逻辑。
  • sendSocketData() 发送数据。

此方法会处理网络连接状态的变更，或者进行数据分发。

对于最重要的 GameFrameEngine 这个实现了大部分的游戏框架逻辑。

可以看到，对于游戏消息，类型为 yl.MDM_GF_GAME 都是交给了 GameFrameEngine 中的 字段 _viewFrame 来处理。从字段意义上来看，这表示了视图的数据模型。

默认情况下，在 BaseFrame:ctor(view,callback) 中会将 _viewFrame 设置为 view。

但是对于 ClientScene 来说，游戏房间的展示，游戏的逻辑等，都不是一个新的 Scene 而是一个 Layer ，所以会动态的设置 _viewFrame，实现将不同的视图模型逻辑处理。
