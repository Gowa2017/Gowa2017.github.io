关于mermaid.cli渲染成图片文件的过程

Posted on 2020-01-11 In JavaScript Disqus:

根据 mermaid 官方文档的说明， mermaid 实际上分成两个部分。mermaid.js 是作为网页集成模块，mermaidAPI 是用来进行定制更多细节的，事实上 mermaid.js 就使用了 mermaidAPI 来进行一些默认的设置。最核心的，其实还是 mermaidAPI 的 render() 方法，其会将我们定义的图表渲染成 svg 代码。

mermaid.cli

mermaid.cli 是 mermaid.js 的一个命令行接口，其原理是利用了一个 headless 的 Chrome node.js API puppeteer 来进行模拟的浏览器操作。

其本质上是用模拟的浏览器 pupeteer 来加载 mermaid.min.js ，对定义的图形进行渲染成 svg 代码在浏览器中。不过是有 bug 存在的.

然后，根据我们的想要的格式，进行转换：

  • 如果是需要的 svg 格式，直接输出网页中的 svg 代码即可
  • 如果需要的是 png 格式，那么就利用了 模拟浏览器的截图功能
  • 如果是 pdf 格式，就利用了浏览器的保存为 pdf 功能。

我想要的是如下效果

但是渲染的结果却是 两个框都是黑的，箭头只剩下了尾部，找了一下谷歌上面看到很多。

基本上可以确实是 mermadi.cli 的问题：

  • https://github.com/mermaid-js/mermaid/issues/664
  • https://github.com/jgm/pandoc/issues/5126
  • https://gitlab.gnome.org/GNOME/librsvg/issues/428 提供的解决办法

究其原因，可能是因为 mermaid.cli 输出的 svg 格式并不是很规范的，所以无法支持。其将 style 写在 svg 标签内，但是并不会被识别。
