2019-11-04

为了规划好自己的时间，规划管理好项目，所以对于如何管理自己进行一下整理。

个人事项的管理，按照 GTD 的理论来，就使用 OmniFocus 这个工具。

OmniFocus GTD

  • inbox 待整理的下一个动作
  • project 有相关性的动作组成的一个集合
  • perspective 透视。根据自己的需求，一览整合后的动作列表

OF 对于一个动作，提供的属性不多大概有：结束日期，推迟日期，持续时间，状态，所属工作区，所属项目这几个属性，肯定远远不够的，所以又增加了标签功能，用标签来进行增加透视的维度。

而我最理想的情况是，对于一个 Work 透视，我能看到 Work 工作区下面所有项目，根据时间，项目分组后的列表，同时还能切换 根据项目，时间来分组的列表

过滤器设置：

  1. 归属 work 工作区
  2. 拥有 active 的 动作 (很遗憾，不提供这个过滤选项）
  3. 通过 project [due] 进行分组
  4. 通过 due [project] 排序

这样我就能看到当天我该干的事情，或者某个项目该干什么事情。

OmniPlan

这个主要是用来做项目进度跟踪，人员管理，计划，已经报告的。我比较看好的是他的基线对比，任务报告，已经甘特图的生产。

配合这几个使用的时候，会造成困扰。主要再于，OF 与这个 OP 都很多概念上都有相似的地方。而作为项目跟进的话我需要跟进的事项都是比较多的。究竟是在 OP 里面进行跟进，还是在 OF 里面进行跟进，感觉重复了。

最终做了一个强制性的改变。OP 重要的是进行管理与进度，计划的的按期执行，而 OF 重在的执行，把立马能做的事情立刻就进行完成。

最终，用 OP 来进行计划，OF 用来执行。如果是真正单纯的管理，不用跟进太多内容的话，就不会有这样的困扰了。

最终

在 OP 上进行计划 哪些任务是需要做的，什么时间，谁来完成。

对于 OP 上的每个任务，在 OF 上拆分成为动作，定好节点。

有一个工具 https://felixchenier.uqam.ca/focusplan/ 可以用来进行双向数据同步

其需要在 OP 中设置 “End No Later Than 约束，才会出现截止日期。

工作流程如下：

  1. 在某日，我们在 OP 打开项目， PullFromOmniFocus 将所有的任务都同步过来，但是任务的完成却不会过来。
  2. 在 OP 中进行任务的计划，变更，修改等，然后 PushToOmniFocus 则会将内容同步过去，包括完成状态。
